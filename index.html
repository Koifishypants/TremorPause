Connection Troubleshooting Why "Connect" might not find your Arduino: Python uses a direct MAC address (BleakClient), which Web BLE on iOS cannot do. The standard "Connect" button filters by service UUID — the Arduino must advertise that UUID in its BLE packet.

Best fix: Enter your Arduino's BLE device name above (check your Arduino sketch, it's set with BLE.setLocalName("…")), then hit Apply All. The button will filter by name instead.

Alternative: Use Scan all BLE devices — it shows every nearby device; pick yours from the list. Service access still works because the service UUID is passed as optionalServices.
